## Phase 0.15.2 — Medium dedupe (APIs stable)

### 4. REST schemas vs `CqrsSchemaRegistry`

**Open item from 0.15 plan:**

> Confirm `rest/schemas.py` duplicates removed where registry replaces them

**Today:** REST handlers validate via `rest/schemas.py`; services validate via `CqrsSchemaRegistry`. Two sources of truth.

**Safe path (experimental — migrate directly, no “deprecated until 0.16” markers):**

1. Audit overlap — map `WIZARD_INPUT_BODY`, `PROVIDE_INPUT_BODY`, `ListJobStatusQuery` shapes, etc. to registry entries
2. Add `rest/schema_bridge.py` — project command schemas onto HTTP bodies (strip path params)
3. REST `validate_body()` → `body_schema_for_command(ctx.schemas, CommandType, properties=(...))` where types align
4. **Keep** `rest/schemas.py` for HTTP-only concerns: multi-variant bodies (`SUBMIT_WIZARD_BODY`, `VALIDATE_FLOW_BODY`), `openapi_components()`, variant-error helpers, response shapes (`SNAPSHOT_SUMMARY`, …)
5. Delete deduped constants — do **not** leave deprecation shims

### 5. Serializer import consolidation

`flow_summary` / `process_summary` live in `palm.common.services.views`; `serializers.py` re-exports them.

- Update callers to import `views` directly
- **0.15.2:** migrate imports
- **0.15.3:** trim `serializers.py` to snapshot-only (`snapshot_summary`, `snapshot_detail`)

### 6. MCP dual-backend clarity

Both paths intentional — **do not delete**:

| Mode | Backend |
|------|---------|
| `PALM_MCP_IN_PROCESS=1` | `PalmInProcessBackend` |
| `PALM_MCP_IN_PROCESS=0` | `PalmRestClient` |

Cleanup = tests + docs:

- `test_mcp_in_process.py` + `test_mcp_tools.py` explicitly cover both modes
- Optional slow integration: `PALM_MCP_IN_PROCESS=0` against `just palm-server`

**Plan:** [2026-06-30-0.15.2-dedupe.md](../plans/2026-06-30-0.15.2-dedupe.md)

## Phase 0.15.3 — Legacy removal (was 0.16 in early notes)

**No migration window.** Delete in-repo; update callers in same release.

### 7. Wizard-named aliases in `palm.common`

```102:106:src/palm/common/interactive_runtime.py
# Deprecated aliases — prefer interactive_* names.
resolve_wizard_job = resolve_interactive_job
provide_wizard_input_for_instance = provide_interactive_input_for_instance
```

**Action:** grep callers → already on `interactive_*` in-tree → **delete aliases** + `previous_wizard_step`, `ChildWizardCompletionHook`, `create_cli_app()`.

**Do not** write `MIGRATION-0.16.md` deprecation schedule for internal APIs. Note removals in `CHANGELOG [0.15.3]` only.

### 8. `SubmitWizardCommand` vs `SubmitFlowCommand`

**Keep** `SubmitWizardCommand` — pattern-owned CQRS routing, not noise. Execution uses `SubmitFlowCommand` + `flow_command_from_body`.

### 9. `archive/`

Per AGENTS.md: never import from `archive/`. Audit only:

```bash
rg "from archive|import archive" src/ tests/
```

If clean, no action. Do not delete `archive/` without ADR.
