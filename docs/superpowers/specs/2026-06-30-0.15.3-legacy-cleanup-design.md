# 0.15.3 Design: Legacy Removal

**Status:** Approved  
**Date:** June 30, 2026  
**Version target:** 0.15.3  
**Theme:** Delete transitional aliases and shims. No deprecation window.

**Parent track:** [0.15 cleanup track](2026-06-30-0.15-cleanup-track-design.md) — ruff + 0.15.0 tag (hygiene), dedupe (0.15.2), **legacy removal (this doc)**.

---

## Policy

Palm is **experimental**. There is no migration period for internal API cleanup:

- Remove deprecated aliases and shims in **0.15.3**, not 0.16.
- Do not add new `DeprecationWarning` wrappers or `MIGRATION-0.16.md` schedules — delete and update callers in-repo.
- Breaking changes are acceptable when they reduce duplication.
- Public REST URLs and MCP tool names stay stable.

**Prerequisites:** 0.15.0 tagged; 0.15.2 REST schema dedup + serializer import migration complete.

---

## Problem

0.15 introduced `CqrsSchemaRegistry` and services, but transitional artifacts remain:

1. **Wizard-named aliases** in `interactive_runtime.py` (`resolve_wizard_job`, etc.) — superseded by `interactive_*` names used everywhere in-tree.
2. **`create_cli_app()`** — deprecated wrapper; no in-repo callers; `create_cli_host()` is canonical.
3. **`ChildWizardCompletionHook`** — alias of `ChildCompletionHook`; exported but unused.
4. **`serializers.py` re-export shim** — after 0.15.2 import migration, only `snapshot_*` should remain.
5. **`ssr/explorer/collection_input.py`** — one-line re-export of `resolve_wizard_form_input`.
6. **Docs** — `MIGRATION-0.10.md` / `ARCHITECTURE.md` still mention `create_cli_app`.

---

## Goals

| Goal | Outcome |
|------|---------|
| Delete aliases | No wizard-prefixed interactive_runtime aliases; no `create_cli_app`; no `ChildWizardCompletionHook` |
| Slim REST layer | `serializers.py` snapshot-only; delete `collection_input.py` shim |
| Docs honesty | Remove deleted APIs from MIGRATION/ARCHITECTURE; CHANGELOG `[0.15.3]` lists removals |

## Non-goals (0.15.3)

- Definition catalog write paths (`DefinitionService.save_flow`, etc.)
- Full OpenAPI generation from registry (partial dedup only)
- WebSocket surface
- Removing CLI `wizard start` user command (intentional UX shortcut, not a code alias)
- Removing `PalmRestClient` (still required for remote MCP)
- Pydantic CQRS migration

---

## Architecture

REST schema dedup ships in **0.15.2** ([dedupe plan](../plans/2026-06-30-0.15.2-dedupe.md)). This release only deletes aliases and remaining shims.

### Alias removal map

| Remove | Replace with |
|--------|--------------|
| `resolve_wizard_job` | `resolve_interactive_job` |
| `require_wizard_job` | `require_interactive_job` |
| `provide_wizard_input_for_instance` | `provide_interactive_input_for_instance` |
| `request_wizard_backtrack_for_instance` | `request_interactive_backtrack_for_instance` |
| `previous_wizard_step` | `previous_interactive_step(..., pattern="wizard")` |
| `create_cli_app()` | `create_cli_host()` |
| `ChildWizardCompletionHook` | `ChildCompletionHook` |
| `serializers.flow_detail` etc. re-exports | `palm.common.services.views` |
| `explorer/collection_input.resolve_wizard_form_input` | `palm.common.operator.collection_input` |
