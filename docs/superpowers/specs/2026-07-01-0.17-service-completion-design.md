# 0.17 Design: Complete the Service-Domain API

**Status:** Approved (July 1, 2026)  
**Version target:** 0.17.0 – 0.17.3 (0.18.0 WebSocket scoped separately)  
**Builds on:** [0.16.5 shipped](VISION-0.16.md) — services are the API; wizard/catalog legacy removed  
**Prerequisite:** `palm/services/` extraction, per-domain REST/MCP, `MIGRATION-0.16.md`

---

## Problem

0.16.5 made `palm/services/` the product API, but **transport is only half-migrated**:

| Gap | Symptom |
|-----|---------|
| **System REST incomplete** | `SystemService` has `inspect_job`, `list_instances`, `cancel_job`, snapshots — REST exposes only `doctor`, `list_jobs`, `get_job` |
| **Legacy monolith routes** | `/v1/jobs`, `/v1/instances`, `/v1/plans`, `/v1/snapshots` still on `route_table.py` |
| **Process execution stub** | `ProcessExecutionService` empty; plans API is the only process submit path |
| **Edge consumers stale** | `PalmRestClient` (MCP proxy), palm provider remote client, Explorer copy still reference legacy paths |
| **OpenAPI/docs blind** | `/v1/openapi.json` generated from `route_table.py` only — `/v1/api/*` invisible |

Integrators and compositional flows (palm provider) therefore still depend on transport-shaped URLs 0.16 was meant to replace.

---

## Goal

**Finish 0.16's deferred work** without adding new product domains:

1. **System service** owns all observe/lifecycle HTTP under `/v1/api/system/…`
2. **Process execution service** owns multi-flow runs under `/v1/api/processes/…`
3. **Delete** legacy monolith job/instance/plan routes (breaking, experimental policy unchanged)
4. **Align consumers** — MCP REST proxy, palm provider remote, Explorer hints
5. **Unified OpenAPI** from per-service route registries

**No `palm/services/palm/`** — the palm provider remains a `palm/providers/palm/` plugin consuming existing service domains.

---

## Principle (unchanged from 0.16)

```
User / Agent / Explorer / CLI / Palm provider
        ↓
palm.runtimes.*          ← thin transport per service domain
        ↓
palm.services.*          ← definitions | execution/* | system
        ↓
palm.common.cqrs + patterns/providers
        ↓
palm.core.*
```

---

## Policy

| Rule | 0.17 |
|------|------|
| Breaking changes | Allowed — delete `/v1/jobs`, `/v1/instances`, `/v1/plans` when service parity exists |
| Deprecation shims | None — update `MIGRATION-0.17.md` instead |
| Job-level `POST …/input` | **Removed** — interactive input is `execution/flows` session verbs only |
| `POST /v1/jobs` (submit) | **Removed** — flow start is `POST /v1/api/flows/{flow_id}/create` |
| Core purity | Unchanged |
| `palm/common/services/` | Still `BaseService` + `errors` only |

## 0.17.0 — System service REST

### Service methods (already exist on `SystemService`)

| Method | Purpose |
|--------|---------|
| `doctor` | Engine health |
| `list_jobs` | Paginated job board |
| `get_job` | Job status by id |
| `inspect_job` | Rich context (events, wizard state, next actions) |
| `cancel_job` | Cancel non-terminal job |
| `list_instances` | Instance index |
| `inspect_instance` | Pattern-aware instance view |
| `list_snapshots` | Snapshot list for instance |

### New REST surface (`/v1/api/system/…`)

| Method | Path | Handler delegates to |
|--------|------|----------------------|
| `GET` | `/v1/api/system/doctor` | `ctx.system.doctor` *(exists)* |
| `GET` | `/v1/api/system/jobs` | `ctx.system.list_jobs` *(exists)* |
| `GET` | `/v1/api/system/jobs/{job_id}` | `ctx.system.get_job` *(exists)* |
| `GET` | `/v1/api/system/jobs/{job_id}/context` | `ctx.system.inspect_job` |
| `POST` | `/v1/api/system/jobs/{job_id}/cancel` | `ctx.system.cancel_job` |
| `GET` | `/v1/api/system/instances` | `ctx.system.list_instances` |
| `GET` | `/v1/api/system/instances/{instance_id}` | `ctx.system.inspect_instance` |
| `GET` | `/v1/api/system/instances/{instance_id}/tree` | existing `build_invoke_tree` helper via instances handler |
| `GET` | `/v1/api/system/instances/{instance_id}/snapshots` | `ctx.system.list_snapshots` |
| `GET` | `/v1/api/system/instances/{instance_id}/snapshots/{snapshot_id}` | snapshot get (reuse `handlers/snapshots.py` logic) |
| `POST` | `/v1/api/system/instances/{instance_id}/resume` | resume helper (reuse `handlers/instances.py` logic) |

### Intentionally not on system

| Legacy route | Replacement |
|--------------|-------------|
| `POST /v1/jobs` | `POST /v1/api/flows/{flow_id}/create` |
| `POST /v1/jobs/{id}/input` | `POST /v1/api/flows/{flow_id}/session/{session_id}/input` |
| `POST /v1/plans/*` | 0.17.1 process routes |

### Registry update

Extend `services/system/registry.py` `ObserveVerb` entries to match mounted routes (documentation + future OpenAPI).

### Consumer updates (0.17.0)

| Consumer | Change |
|----------|--------|
| `runtimes/mcp/rest_client.py` | `list_waiting_jobs`, `get_job_context`, `cancel_job`, tree/snapshots → `/v1/api/system/…` |
| `common/job_context.py` | `next_actions` paths → flows session input |
| Explorer SSR copy | `/v1/jobs` → `/v1/api/flows/…/create` or system paths |
| `doc_examples.py` | Legacy example paths only for routes that remain |

### Monolith shrink

After parity: `route_table.py` retains **meta only** (`/health`, `/v1/docs`, `/v1/openapi.json`). Job/instance/plan/snapshot route definitions removed. `handlers/jobs.py`, `instances.py`, `snapshots.py` remain as **implementation** called from `system/handlers.py` (not deleted until handlers inlined or re-exported cleanly).

## 0.17.1 — Process execution service

### Behavior

Process submit today: `PreparePlansCommand` → plan ids → `SubmitPlansCommand` → jobs. This moves behind `ProcessExecutionService` with a **staged REST** shape matching MCP `palm_processes_submit` (prepare + submit in one tool call, two HTTP steps internally).

### Service API

```python
class ProcessExecutionService(BaseService):
    def prepare(
        self,
        process_id: str,
        *,
        by_id: bool = False,
        job_id: str | None = None,
        body: dict[str, Any] | None = None,
    ) -> dict[str, Any]: ...

    def submit(self, plan_ids: list[str]) -> dict[str, Any]: ...

    def run(
        self,
        process_id: str,
        *,
        by_id: bool = False,
        job_id: str | None = None,
        body: dict[str, Any] | None = None,
    ) -> dict[str, Any]:
        """prepare + submit — convenience for MCP and palm provider."""
```

Implementation composes existing `runtime.executor.submit_process` / plan registry via CQRS (same as `ApplicationHost.submit_process` and `_prepare_plans`).

### REST (`/v1/api/processes/…`)

| Method | Path | Maps to |
|--------|------|---------|
| `POST` | `/v1/api/processes/{process_id}/prepare` | `prepare` |
| `POST` | `/v1/api/processes/submit` | `submit` (`{"plan_ids": [...]}`) |
| `POST` | `/v1/api/processes/{process_id}/run` | `run` (optional convenience) |

### Registry

Populate `services/execution/processes/registry.py` with `ProcessCommandSpec` entries (mirror `flows/registry.py` pattern).

### Host wiring

`host.execution.processes` already exists on `ExecutionCoordinator` — implement dispatch grammar:

```
["processes", process_id, "prepare"]
["processes", "submit"]  + params plan_ids
["processes", process_id, "run"]
```

### MCP

`PalmInProcessBackend.prepare_plans` / `submit_plans` → call `host.execution.processes` instead of raw REST paths.

## 0.17.2 — Palm provider remote alignment

**Depends on:** 0.17.0 + 0.17.1

### `providers/palm/flow/remote/client.py`

| Function | New path |
|----------|----------|
| `submit_flow_remote` | `POST /v1/api/flows/{flow_id}/create` |
| `get_job_remote` | `GET /v1/api/system/jobs/{job_id}` |
| `submit_process_remote` | `POST /v1/api/processes/{process_id}/prepare` + `POST /v1/api/processes/submit` (or `/run`) |
| `invoke_resource_remote` | *(unchanged — already 0.16)* |

`remote_job_payload` must accept `session_id` from create response and map to `instance_id` field for wait metadata compatibility.

### Docs

`docs/PROVIDER-APPS.md` — remote binding table lists `/v1/api/…` only.

## 0.17.3 — OpenAPI from service registries

### Aggregator

New module `runtimes/server/surfaces/rest/openapi_registry.py`:

- Collect `RouteEntry` tuples from:
  - `definitions/routes.py`
  - `execution/flows/routes.py`
  - `execution/providers/routes.py`
  - `execution/processes/routes.py`
  - `system/routes.py`
- Merge with meta routes (`/health`, `/v1/docs`, `/v1/openapi.json`)
- Feed `openapi.py`, `docs.py`, `doc_examples.py`

### `route_table.py` fate

Becomes **meta-only** or deleted; `rest_routes()` reads aggregator.

### MCP resource

`palm://openapi` reflects full `/v1/api` surface.

## Alternatives considered

| Alternative | Why not |
|-------------|---------|
| Keep `/v1/jobs` as permanent alias | Perpetuates transport-shaped API 0.16 rejected |
| Single `POST /v1/api/processes/{id}/run` only | MCP and tests already two-step; staged REST is honest and matches CQRS |
| `palm/services/palm/` service | Violates domain-boundary model; provider is edge plugin |
| Generic OpenAPI framework in 0.17 | Manual registry merge is enough; framework deferred |
| Migrate job input to system | Flows session REPL is the correct domain |
