# 0.15.2 Dedupe Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Single source of truth for CQRS-aligned REST validation; consolidate serializer imports; clarify MCP dual-backend tests — public REST/MCP contracts unchanged.

**Architecture:** `rest/schema_bridge.py` projects `CqrsSchemaRegistry` command schemas onto HTTP bodies. Handlers use `ctx.schemas`. Deduped constants removed from `rest/schemas.py` (no deprecation shims). Serializer callers import `views` directly. MCP tests document both in-process and HTTP proxy modes.

**Tech Stack:** `CqrsSchemaRegistry`, `DictStateSchema`, pytest, `just guard-common`.
